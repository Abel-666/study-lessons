📦 第三课：TCP/IP — 数据是怎么传输的？

📮 先讲个快递的故事：

你在淘宝下单买一本书：

  1. 你把地址写给卖家（对方知道你在哪）
  2. 卖家把书打包成包裹（数据分片）
  3. 快递员按地址送到你家（路由传输）
  4. 你拆开包裹，确认是你要的书（重组数据）
  5. 你跟卖家说"收到了"（确认回执）

TCP/IP 就是互联网世界的快递系统。

一、IP = 地址系统

你家的地址是"XX省XX市XX街道XX号"，服务器的地址就是 IP地址。

IPv4 长这样：192.168.1.1（4组数字，每组0-255）

IPv6 长这样：240e:390:771:...（更长，彻底解决地址不够用的问题）

特殊的 IP：

  • 127.0.0.1 = 本机（你自己这台电脑）
  • 192.168.x.x = 内网地址（你家路由器分配的）
  • 你访问微信时，域名 weixin.qq.com 会被 DNS 解析成 IP 地址

二、TCP = 传输协议（保证数据不丢不乱）

TCP 干的活儿就三个字：不丢包。

TCP 三次握手（建立连接）：

第一次握手 👋
你 → 微信： "我要连你"（SYN）

第二次握手 👋
微信 → 你： "收到，我准备好了"（SYN-ACK）

第三次握手 👋
你 → 微信： "好，开始传输！"（ACK）

这下双方确认对方还活着，通道建立成功。

就像你和对讲机通话：

  • "你能听到吗？"
  • "能听到你说！"
  • "好，我开始说了！"

——三次确认，这就是三次握手。

三、端口 = 哪个应用在通信

IP地址找到的是某台电脑，那这台电脑上那么多应用（微信、浏览器、游戏）谁收数据？——端口号决定。

就像一栋大楼（一台电脑）里的不同房间号：

  • 80 — HTTP 默认端口
  • 443 — HTTPS 默认端口（你刚学的加密传输）
  • 22 — SSH（远程连接Linux）
  • 3306 — MySQL 数据库
  • 53 — DNS（域名解析）

所以访问一个服务 = IP地址:端口号

https://weixin.qq.com:443 → 找微信服务器，走443端口（HTTPS加密）

💡 端口为什么重要？—— 黑客扫描端口就是在找你电脑上"开了哪些门"，每个门都可能是漏洞。这就是为什么安全工程师要把不该开的端口关掉。

四、数据包的一生（完整路径）

你在微信发一条消息"你好"，这一路上发生了什么：

你的手机 → 你的消息"你好"
↓
① 应用层（HTTPS加密，变成一堆乱码）
↓
② 传输层（TCP切分成小包，编号，加上端口号）
↓
③ 网络层（加上目标IP地址：微信服务器的IP）
↓
④ 网络接口层（转成电信号，从WiFi/4G发出去）
↓
🚀 穿过路由器、交换机、海底光缆……
↓
⑤ 微信服务器收到 → 逐层拆包解密 → 显示"你好"

五、DNS — 域名是如何变成IP的？

你记得住 weixin.qq.com，但你记不住它的IP地址。

DNS（域名系统） 就是互联网的电话本——你把域名给它，它告诉你对应的IP。

你在浏览器输入 weixin.qq.com →

系统问DNS服务器："weixin.qq.com 的IP是多少？" →

DNS 返回对应IP地址（比如 58.250.137.36） →

浏览器再用这个IP去连接微信服务器

💡 DNS 安全小知识： 有一种攻击叫 DNS 劫持——骗子篡改 DNS 记录，让你访问假的网站还以为是真的。 所以安全工程师会检查 DNS 配置是否正确，或者用 DNSSEC 来防止篡改。
