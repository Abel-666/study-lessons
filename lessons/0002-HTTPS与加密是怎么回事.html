🔐 第二课：HTTPS 与加密是怎么回事

一、先搞懂 HTTP 和 HTTPS 的区别

HTTP = 裸奔版
你给网站发消息，内容是明文。中间任何一个人都能看到你发了什么。

HTTPS = 加密版
你给网站发消息，内容加了密。中间人截获了也看不懂。

那个多出来的 S 代表 Secure（安全），它的底层就是 TLS/SSL 加密协议。

📮 类比时间：

HTTP = 你写在明信片上寄出去，邮递员翻过来就能看到内容。

HTTPS = 你装在密码箱里寄出去，只有收件人知道密码，邮递员拿到也打不开。

二、加密的两种方式

🔑 对称加密（一把钥匙）

加密和解密用同一把钥匙。

你给朋友一个密码箱，你们俩各有一把同样的钥匙。你锁进去，他打开看。

问题：钥匙怎么安全地给对方？如果邮寄钥匙的时候被人复制了怎么办？

🔓 非对称加密（两把钥匙）

每个人都有一个 公钥（公开的）和一个 私钥（自己藏好的）。

  • 公钥加密的 — 只有对应的私钥能解开
  • 私钥加密的 — 只有对应的公钥能解开

你放一个密码箱在门口，任何人都可以把信放进去锁好（公钥加密），但只有你有钥匙能打开（私钥解密）。

三、HTTPS 握手的真实过程

当你打开一个 HTTPS 网站时，背后发生了这些事（3步搞定）：

第1步：打招呼
你的浏览器说："嗨，我要连你，支持加密吗？"
服务器说："支持！这是我的证书（公钥），你看一下。"

第2步：验证身份
浏览器检查服务器给的证书是不是真的——
相当于你检查对方的身份证是不是公安局发的。
这个发证机构叫 CA（证书颁发机构）。

第3步：协商密码
你和服务器的用非对称加密安全地商量出一个"临时密码"
之后的对话都用这个临时密码进行 对称加密——又快又安全。

💡 一句话总结：HTTPS = 非对称加密用来安全地传递对称加密的钥匙，之后用对称加密来加密整个对话。

四、怎么验证一个网站是安全的？

下次打开登录页面时，看看浏览器地址栏：

  • ✅ 🔒 https:// + 锁头图标 → 安全，可以输密码
  • ⚠️ http:// 没有锁头 → 不安全，不要输任何敏感信息
  • ❌ 锁头上有红叉或警告 → 可能有中间人攻击，不要继续操作

进阶操作：点击地址栏的锁头 → 查看证书信息 → 能看到是谁颁发的证书
